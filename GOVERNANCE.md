# Governance

Maestro uses a maintainer-led, consensus-seeking model.

## Roles

- Contributors submit issues, documentation, adapters, tests, and code.
- Maintainers review and merge changes, manage releases, and handle security reports.
- Committers are maintainers trusted to merge within designated areas.

Maintainer and committer status is earned through sustained, technically sound, respectful contributions. Changes to project governance require public discussion.

## Decision making

Routine changes require maintainer review. Public API, workflow compatibility, security model, licensing, and governance changes require an ADR and approval from at least two maintainers when the project has two or more active maintainers.

Commercial users receive no special merge rights. Enterprise requirements are welcome when they preserve a generally useful core and do not disadvantage community users.

## Releases

Releases follow semantic versioning and use signed tags when release infrastructure supports them. Release notes must identify public API changes, workflow migration requirements, and security-relevant changes.
